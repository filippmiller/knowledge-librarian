### 3. Email/chat knowledge mining pipeline (THIS is the active work)
All scripts in `scripts/email-mining/` (read `.env` for the webhook; NO hardcoded secrets). Run with `node scripts/email-mining/<x>.mjs` (use PowerShell on this machine ‚Äî bash PATH glitches intermittently).

**Pipeline (final, correct approach):**
1. `build-threads.mjs <from> <to> <ownerType>` ‚Äî fetch+clean CRM email threads. `ownerType` 2=Deal, 1=Lead. Writes `scratchpad/threads-<deal|lead>-<from>.json`.
2. `build-chats.mjs <YYYY-MM>` ‚Äî fetch open-line CHAT transcripts (Telegram/WhatsApp/VK) via `imopenlines.session.history.get`. Writes `scratchpad/chats-<month>.json`. Has retry/backoff for transient sockets.
3. **Extraction** = dispatched to **subagents (Claude/Sonnet, NOT an API)** reading `scratchpad/EXTRACT_RULES.md` + a thread file slice ‚Üí write `scratchpad/kn-*.json`. Rules: extract reusable KNOWLEDGE (capability/requirement/process/policy/location/pricing_policy), DROP status/logistics/one-off numbers, generalize, Russian output, prices‚Üípolicy not sums.
4. `aggregate.mjs` ‚Äî merges all raw items ‚Üí `scratchpad/all-raw-knowledge.json` + fetches `scratchpad/price-catalog.json` (28 services, real prices).
5. Split by type (`scratchpad/by-type/*.json`) ‚Üí per-type dedup subagents ‚Üí `scratchpad/dedup/*.json`.
6. `assemble-final.mjs` ‚Äî combines dedup files + price catalog ‚Üí `scratchpad/knowledge-base-final.{md,json}`.

**May 2026 result:** 1275 deal-emails + 1376 lead-emails + 130 chats ‚Üí **406 raw knowledge items ‚Üí 180 canonical rules** (capability 56, process 47, policy 29, requirement 22, pricing_policy 15, location 11). Copied to tracked `docs/email-bot/`.

## PROBLEMS HIT (so the next agent doesn't repeat them)
- **Bash PATH glitches** intermittently (`node`/`grep` "command not found"). Use **PowerShell** for `node` runs; use Grep/Glob tools not shell grep.
- **Bitrix socket timeouts** on long sequential fetch loops (`UND_ERR_SOCKET`, connect timeout). `build-chats.mjs` has retry/backoff; add the same to any new fetch loop. Just re-run on failure.
- **Subagent API flakiness** ("Stream idle timeout"/"socket closed") on LARGE single tasks (one agent merging 406 items timed out twice). Fix: **split work small** (per-type, ~30-80 items each) ‚Äî those succeeded.
- **`scratchpad/` is gitignored** (`.gitignore:67`). All raw/intermediate mining files live there and will NOT push. Deliverables were copied to `docs/email-bot/`. To regenerate scratchpad, re-run the pipeline.
