## GOAL

The Avrora Telegram bot (@avroratranslatebot) answers questions about apostille /
document legalization from a knowledge base built out of uploaded documents. Two
problems drove this session:

1. **Trust/transparency** — the bot wasn't always confident or correct, and it was
   unclear whether an answer came from the uploaded documents or from the LLM's
   general knowledge.
2. **Self-improvement** — when the bot is uncertain, it should still answer from
   general AI knowledge, BUT turn that answer into a draft question→answer rule and
   send it to the admin for approval/correction. On approval it's saved to the KB so
   the next identical question is answered confidently from the base.

Both are **DONE and deployed**. This handoff is to (a) let a fresh session continue
the remaining content work, and (b) act on Codex's review once it comes back.

## ARCHITECTURE & FLOW

```
question → classifyScenario() [gate BEFORE retrieval]
   ├ scenario_clear → retrieve within scenario
   ├ needs_clarification → ask (sc:<id> buttons)   ← not an answer
   ├ knowledge_lookup → general retrieval
   └ out_of_scope → general_ai / "нет данных"
→ hybrid retrieval (semantic+keyword RRF): Step5 rules, Step6 qaPairs (both per-term prefilter)
→ LLM compose → verifyAnswer() strips unsupported claims
→ confidence = bestSemanticScore + coverageScore*0.1
→ if low-trust (general_ai|low|insufficient & not clarification):
      escalateUnconvincingAIAnswer → createKnowledgeGapSuggestion → AIQuestion(knowledge_gap,OPEN)
         → super-admins get ✅/✖️ buttons (kg:approve:<id>/kg:reject:<id>)

APPROVAL: super-admin taps ✅ (TG) or web PATCH {action:'approve'}
   → approveKnowledgeGap: guard issueType+status==OPEN → create ACTIVE QAPair
     {metadata:{origin:'ai-suggested',approvedBy,approvedAt,fromAIQuestion}} → AIQuestion→ANSWERED
   → next identical question retrieves the new QAPair → answerSource=knowledge_base
```

## PHILOSOPHY & DECISIONS

| Decision | Rationale | Rejected |
|----------|-----------|----------|
| Human ALWAYS gates the loop | AI draft answer can be wrong; never auto-save | auto-saving high-confidence general_ai |
| Scenario gate BEFORE retrieval | one clarifying Q beats a confidently-wrong answer | retrieve-then-guess |
| Per-term keyword prefilter + recent-N fallback | exact `contains` recall for Cyrillic terms; `take:100` alone dropped fresh rows | pure semantic top-K |
| `[а-я]*` not `\w*` in RU regexes | JS `\w` excludes Cyrillic — silent miss | `\w*` (the bug we hit twice) |
| Deterministic eval (no LLM judge) | cheap, stable regression gate | LLM-as-judge (flaky, costly) |
| Raw ALTER not db push | db push would drop DocumentRevision | `--accept-data-loss` (forbidden) |
| Test prod via Node fetch, never curl | Windows Git Bash curl corrupts Cyrillic UTF-8 → bot gets "???" → false out_of_scope | curl with `--data` |
